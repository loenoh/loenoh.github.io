Hexo 구글애드센스(Google Adsense) 추가

Feb 07, 2018 in hexo

Hexo 블로그에 Google Adsense를 추가해볼려고 합니다.

구글애드센스를 추가한다는 것은 즉, 블로그에 광고를 삽입하겠다는 뜻입니다.

마찬가지로 광고삽입 전 구글애드센스에 가입이 되어야하고 가입과정은 생략하겠습니다.

사이트 : 구글애드센스(Google Adsense)


전체적인 작업진행

  1. Google Adsense 광고 만들기
  2. 코드 생성
  3. 광고가 나오될 위치에 해당 코드 삽입


구글애드센스(Google Adsense) 광고 만들기

구글애드센스 사이트에 로긴 후 홈으로 이동합니다.

내광고 > 콘텐츠 > 광고단위 > 새광고단위 를 클릭해서 광고를 만듭니다.


텍스트 및 디스플레이 광고를 선택합니다.


이름, 광고크기, 광고유형등을 선택하고 저장 및 코드생성을 클릭합니다.

광고크기는 특별한 경우가 아니면 페이지에 따라 변화되는 반응형을 추천합니다.




코드가 생성된 팝업 화면이 나타납니다.

코드를 복사하셔도 되고 광고단위 목록에서도 코드를 다시 가져올 수 있습니다.



그럼 광고가 만들어졌습니다.

그럼 광고가 나올 곳을 찾아 붙여 보겠습니다.


코드 삽입

hexo의 파일 구조를 보면 테마폴더/layout/layout.ejs 파일이 전체적인 구조에 관련된 파일이고, 세부적인 구조들은 테마폴더/layout/_partial폴더안에 각각 ejs파일로 구성되어져 있다.

그래서 구글 애드센스 코드를 바로 삽입하는 것이 아니라 ejs파일을 생성하고 생성된 파일을 광고가 나오게 될 부분에 삽입하는 형태를 추천합니다.

그럼 테마폴더/layout/_custom_ads 폴더를 하나 생생하고 google_adsense.ejs파일을 생성해서 해당파일에 구글 애드센스에서 생성한 코드를 넣어줍니다.

이제 광고가 삽입될 부분에는 다음과 같은 코드 한줄을 추가하면 적용이 됩니다.


본문 위, 아래 추가

본문에 삽입할 경우는 layout.ejs파일을 수정하면 되고, 파일은 일부만 발췌했습니다.

파일위치 : 테마폴더/layout/layout.ejs



윗 부분을 수정하면 메인 화면 포스팅 위, 아래에도 추가되고 클릭했을 때 해당 포스팅에도 추가되어집니다.
하지만 메인에는 보이지 않고 각각의 포스팅에만 적용할 경우에는 post.ejs파일을 수정하면 가능합니다.

파일위치 : 테마폴더/layout/_partial/post.ejs


사이드바 하단부분 추가

사이드바 하단에 추가는 개인적으로는 보기가 좀 그래서 추가는 안했던 부분인데 필요하신 분은 추가하시면 될 것 같습니다.

수정파일위치와 파일내용은 아래와 같고, 파일내용은 길이 관계로 일부만 발췌했습니다.

파일위치 : 테마폴더/layout/_partial/sidebar.ejs

구글 웹마스터도구(Google Search Console)에 사이트 등록

Jan 30, 2018 in SEO

지금까지 블로그에 Plugin을 이용해서 몇가지 SEO를 적용해 보았다.
이제 구글 웹마스터 도구에 블로그 컨텐츠를 구글봇이 잘 크롤링할 수 있도록 등록해 보자.

사이트 : Google Search Console

크게 아래와 같은 순서로 진행하면 될 듯 하다.

  1. 회원가입
  2. 속성추가
  3. Sitemaps
  4. robots.txt
  5. Fetch As Google
  6. 구글 검색 확인

여기서는 회원가입은 생략하도록 한다.

속성 추가

이제 블로그를 추가할려면 메인페이지에 속성추가를 클릭하면 된다.
사이트 소유권확인하는 페이지로 이동한다.
소유권확인은 HTML 파일 업로드와 다른 대체 방법으로 사이트를 확인해야한다.
주로 소유권확인페이지에서 파일을 다운 받아서 루트에 올려서 확인하는 방법이 보편적이나 구글애널리틱스에 미리 가입을 하신분이면 대체방법 중에 Google 애널리틱스를 선택하고 확인하는 방법도 있다.

참고로 Google 애널리틱스에 미리 등록이 되어져 있으신 분은 그림에서 추적ID를 블로그에 적용하면 된다.

테마 환경설정 파일 _cofig.yml 파일에서 다음 항목에 추가하면 된다.

google_analytics_id: UA-XXXXXXXX-X

기본적으로 네이버블로그나 기타 FTP접근이나 head부분에 소스 수정이 불가한 경우는 소유권확인을 할 수 없다.

소유권확인하는 페이지에 설명이 있지만 좀 더 알아보고자 하시는 분은 구글 사이트 소유권확인에서 확인이 가능하다.

sitemaps

속성을 추가하여 사이트 소유권 확인이 되면 왼쪽 메뉴중 크롤링 > Sitemaps 메뉴에서 이전에 만든 sitemap.xml을 추가하면 된다.
먼저 sitemap추가/테스트부터 해보자.
오른쪽 상단 SITEMAP 추가/테스트 버튼으로 테스트와 추가가 가능하다.

제출 후 바로 반영되지는 않는다. 보통 하루는 걸리는 것 같다.
제출하고 다음날 확인하면 제출된 결과가 그래프로도 나온다.

robots.txt 테스터

sitemap 처럼 작성해둔 robots.txt 파일을 테스트 후 제출해보자

아래 그림에서 처럼 하단부에 파일명을 쓰고 테스트 후 제출하면 된다.

Fetch As Google

구글에서 사이트의 URL을 크롤링하거나 랜더링하는 방식을 테스트 할 수 있는 곳이다.
내 사이트를 잘 가져가고 있는지, 이미지나 스크립트가 봇을 차단하는지 등을 알 수 있다.
꼭 해보시길 권장한다.

메뉴를 클릭해서 url 넣는 곳에 해당 페이지를 넣으면 된다.
넣지 않으면 루트페이지를 가져온다.

가져오기 : 페이지 URL을 가져온다.
가져오기 및 렌더링 : 페이지에 포함된 리소스(이미지나 스크립트)를 렌더링합니다.

버튼을 클릭하면 색인생성요청 이라는 버튼이 나타난다.

색인생성요청을 클릭하면 팝업이 나타나고 원하는 사항을 선택하고 이동을 클릭한다.

그리고 목록을 클릭해서 결과 확인이 가능하다.

크롤링 메뉴에서는 크롤링오류나 크롤링통계도 확인 가능하다.

구글 검색 확인

설정이 마무리 되면 이제 구글봇들이 크롤링을 하게되고 일정 시간이 지나면 구글에 반영이 된다.
구글에 반영이 잘 되고 있는 지 확인은 구글 사이트에서 확인가능하다.

site: [도메인] ex) site:loenoh.github.io

이렇게 치면 구글에서 크롤링된 페이지가 검색이 된다.
